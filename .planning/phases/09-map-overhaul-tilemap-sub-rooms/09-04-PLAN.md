<objective>
Wire the camera limit system into the sub-room transitions and perform the final integration verification of the entire Phase 9 system.

  <action>
Add a helper method to Game.gd that updates camera limits for all currently spawned players:

```gdscript
## Phase 9 (D-03, MAP-07): Updates Camera2D.limit_* on all players to match the current sub-room bounds.
## Called on all peers after every sub-room build (call_local context — no RPC needed).
## Must be called AFTER _current_sub_room_rect_px is set.
func _update_all_camera_limits() -> void:
    for p in get_tree().get_nodes_in_group("players"):
        if is_instance_valid(p) and p.has_method("update_camera_limits"):
            p.update_camera_limits(_current_sub_room_rect_px)
```

Then add a call to `_update_all_camera_limits()` at every site where `_current_sub_room_rect_px` is assigned. There are at least three such sites (from Plan 03 work):

1. In `_ready()`, after `_current_sub_room_rect_px = _first_rect`:
   ```
   _update_all_camera_limits()
   ```
   Note: players may not be spawned yet at _ready() time (they are spawned via `_spawn_all_players()` which is called after _ready). Add a deferred call instead:
   ```gdscript
   call_deferred("_update_all_camera_limits")
   ```

2. In `_transition_to_sub_room()`, after `_current_sub_room_rect_px = rect`:
   ```gdscript
   _update_all_camera_limits()
   ```

3. In `_transition_to_room()`, after `_current_sub_room_rect_px = _trans_rect`:
   ```gdscript
   _update_all_camera_limits()
   ```

Also ensure that Player.gd's `update_camera_limits` method (added in Plan 02) is present. If for any reason it was not added correctly, add it now (same code as Plan 02 Task 2).

Verify that Player.gd does NOT have Camera2D in the MultiplayerSynchronizer's replication config — search for "Camera2D" in the replication_config sub-resource in Player.tscn to confirm it's absent. If it was accidentally added, remove it.
