<objective>
Implement the two core systems that make sub-rooms work: RoomBuilder.gd (the TileMap population engine) and the sub-room state machine in Game.gd.

RoomBuilder.gd reads SUB_ROOM_DATA from RoomLayouts.gd and populates the active room's TileMap node by calling set_cell() for floor, wall, and obstacle rects. It also repopulates spawn point Marker2D nodes with positions from the layout data.

Game.gd gains the sub-room progression state machine: current_sub_room tracking, _transition_to_sub_room() call_local RPC, _open_exit_passage() call_local RPC for host-authoritative gate opening, and _check_sub_room_clear() to detect when all sub-room enemies are dead.

OSMRoomGenerator.gd is deleted. All 3 references to it in Game.gd (lines 89–95 instantiation block, lines 159–163 fetch-on-transition block, lines 454–457 _on_osm_room_ready method) are removed.

This plan has no file overlap with Plan 02 (Player.tscn / Player.gd) — they run in parallel in Wave 1.

Purpose: Without RoomBuilder + the Game.gd state machine, the TileMap nodes from Plan 01 would remain forever empty.

Output: A fully functional sub-room build + progression loop. Running the game as host will load sub-room 1 of Room 1 with tiles placed.
</objective>

Create `scenes/RoomBuilder.gd` with `class_name RoomBuilder` and `extends RefCounted`.

This script has no _ready(). It is instantiated by Game.gd and called when a sub-room must be built.

**Key method: `build_sub_room(room_id: int, sub_room_id: int, game_node: Node) -> Rect2`**

Steps inside build_sub_room:
1. Retrieve layout: `var layout: Dictionary = RoomLayouts.SUB_ROOM_DATA[room_id][sub_room_id]`
2. Get the TileMap node: `var tilemap: TileMap = game_node.get_node("Room%d/TileMap" % room_id)`
3. Clear the TileMap: `tilemap.clear()` — removes all cells from all layers
4. Determine source_id from `layout["tileset_src"]` (SRC_MODERN=0, SRC_DUNGEON=1)
5. Place floor tiles: iterate over `layout["floor"]` array of Rect2i. For each Rect2i(x, y, w, h), nested loop x_off in range(w), y_off in range(h) — call `tilemap.set_cell(0, Vector2i(rect.x + x_off, rect.y + y_off), source_id, layout["floor_tile"])`. For Room 1 (ERBA) apply the grass:asphalt mix rule: every 3rd tile (modulo 3 == 0) uses MC_FLOOR_CRACK atlas coord; every 3rd tile (modulo 3 == 1) uses MC_FLOOR_GRASS_ALT. For Room 2 (Altstadt) every 10th tile places MC_FLOOR_GRASS. Use `(x_off + y_off) % N` as the alternating index. For Room 3, no mixing — pure stone.
6. Place wall tiles: iterate over `layout["walls"]` array of Rect2i, same nested loop, use `layout["wall_tile"]` atlas coord.
7. Place obstacle tiles: iterate over `layout["obstacles"]` array of Rect2i, use `layout["obstacle_tile"]` atlas coord.
8. Update spawn points: Get `game_node.get_node("Room%d/SpawnPoints" % room_id)`. Remove all existing Marker2D children via `for c in spawn_node.get_children(): c.queue_free()`. Then for each position in `layout["spawn_points"]`, create a Marker2D, set its position (already in pixels from RoomLayouts), add it as child of spawn_node.
9. Update enemy spawn points: Same as step 8 but for `"Room%d/EnemySpawnPoints" % room_id` and `layout["enemy_spawns"]`.
10. Store exit tile coords on game_node: `game_node._exit_tile_coords = layout["exit_tile_coords"]` (these are the wall tiles to erase when the exit opens).
11. Calculate and return the sub-room pixel bounds as a Rect2: `return Rect2(0, 0, layout["width_tiles"] * RoomLayouts.TILE_SIZE, layout["height_tiles"] * RoomLayouts.TILE_SIZE)`. Game.gd uses this Rect2 to update camera limits.

**Key method: `build_connector(room_id: int, game_node: Node) -> Rect2`**

Calls `build_sub_room(room_id, 6, game_node)` — connector is sub-room 6 in the data. Returns the Rect2.

**Important implementation notes:**
- `tilemap.set_cell()` in Godot 4.x signature: `set_cell(layer: int, coords: Vector2i, source_id: int, atlas_coords: Vector2i, alternative_tile: int = 0)`. Layer is always 0. Source_id is from layout["tileset_src"]. Atlas_coords is the Vector2i atlas coordinate (not a linear tile index).
- After set_cell() calls, Game.gd is responsible for calling `call_deferred("_bake_navigation")` — do NOT call it from RoomBuilder (separation of concerns; navmesh bake must happen in Game.gd's context).
- The queue_free() on spawn marker children happens immediately — since we're not in a physics frame (this is called from an RPC body), queue_free() is safe. Use `for c in spawn_node.get_children(): c.queue_free()` then add new children. Note: queue_free() marks for deletion but child is gone before next frame — safe to re-add new children of the same name immediately.

**TileMap collision enable/disable:**
Add a helper method: `func set_tilemap_collision(room_id: int, enabled: bool, game_node: Node) -> void`. Gets the TileMap for that room, sets `collision_layer = 1 if enabled else 0` and `collision_mask = 1 if enabled else 0`. This is called by Game.gd's _transition_to_room to handle the hidden-room collision pitfall (Pitfall 4).
