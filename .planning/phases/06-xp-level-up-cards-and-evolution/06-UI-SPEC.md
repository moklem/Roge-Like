| Property | Value |

## Spacing Scale

Pixel values for Godot Control node offsets and margins. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inner label padding, icon gaps |
| sm | 8px | Separation between HUD elements (LevelLabel–XPBar gap) |
| md | 16px | Card panel internal padding; HUD bar margin from screen edge |
| lg | 24px | Gap between card panels in the overlay HBoxContainer |
| xl | 32px | Card overlay container padding from screen center |
| 2xl | 48px | XP bar bottom anchor offset from screen bottom edge |
| 3xl | 64px | Card overlay total height (HBoxContainer minimum height) |

Exceptions:
- XP ProgressBar: `custom_minimum_size = Vector2(0, 12)` — 12px height fits the bottom HUD strip without crowding
- HealthBar (pre-existing): 8px height, unchanged in this phase
- Teammate level-up Label: no padding (world-space Label, matches existing RoleLabel style at `offset_top = -50`)
- Card panel minimum width: 160px (`custom_minimum_size = Vector2(160, 64)`) — fits 3 cards side-by-side at 1280px
- **Stage visual ColorRects (world-space, Node2D children of Player):** offset values are pixel geometry centered on the player origin — non-multiples-of-4 values (±10, ±18, ±22, ±6, ±14px) are geometrically required to achieve the declared pixel dimensions (e.g., a 20px-wide body requires ±10 from center; a 36px-tall body requires ±18). The 4px grid rule applies to UI Control layout (HUD, card overlay), not to world-space sprite geometry on Node2D nodes.
- **LevelUpLabel world-space position:** `offset_top = -70`, `offset_bottom = -54` — 16px-tall label band positioned 54–70px above the player origin. Art-fit values for visual placement above the player character and existing RoleLabel (which sits at offset_top = -50). Non-multiple-of-4 values are justified by the need to avoid overlapping the RoleLabel.

## Keyboard Interaction Map

All input is keyboard-only. No mouse click targets exist in Phase 6 UI.

| Key | Action | Context | GDScript InputMap Action |
|-----|--------|---------|--------------------------|
| `A` | Cycle card selection left (wraps) | Card overlay open, owning peer only | `ui_left` (built-in) |
| `D` | Cycle card selection right (wraps) | Card overlay open, owning peer only | `ui_right` (built-in) |
| `Space` | Confirm selected card | Card overlay open, owning peer only | `ui_accept` (built-in) |
| `Enter` | Confirm selected card | Card overlay open, owning peer only | `ui_accept` (built-in) |
| All other keys | Blocked during card pick | `is_picking_card = true` suppresses all other input on owning peer | Input guard in `Player._unhandled_input` |

**Guard rule:** When `is_picking_card = true`, the owning peer's `Player._unhandled_input` consumes A/D/Space/Enter exclusively and does NOT propagate them to movement or ability systems. No other player input is processed on the owning peer until the card is confirmed.

**Non-blocking guarantee:** Keys pressed by OTHER peers are completely unaffected. The card overlay CanvasLayer is local; `SceneTree.paused` is never set (W4 compliance).

## Interaction Flow Diagram (text)

```
Enemy killed → XpOrb.gd _request_collect → receive_xp RPC to owning peer
  → xp += 5 → XPBar.value updated
  → if xp >= threshold:
      → is_picking_card = true (replicated → LevelUpLabel shows on all peers)
      → is_invulnerable = true (damage guard in receive_damage)
      → CardOverlay.visible = true (local only)
      → input focus: A/D cycles _selected_card index (0/1/2)
          → selected card border turns #66FF66, others revert to #595966
      → Space or Enter pressed:
          → confirm_card_pick RPC sent to host (card index)
          → host validates, broadcasts effect
          → is_picking_card = false (replicated → LevelUpLabel hides on all peers)
          → is_invulnerable = false
          → CardOverlay.visible = false
          → xp = 0, XPBar.value = 0, XPBar.max_value = next threshold
          → LevelLabel text updated to new level
```
