| SUSPENSION lit background | `Color(0.9, 0.8, 0.05, 1)` | #E6CC0D | "SUSPENSION ⚡ IMPACT" indicator active state only |

**Fade behavior colors:** During the 0.5-second fade-out tween, `modulate.a` interpolates from `1.0` to `0.0` on the indicator's ColorRect `bg_color` or on the node's modulate property. At fade completion, restore the indicator to idle state (reset `modulate.a = 1.0`, set bg back to `Color(0.10, 0.10, 0.10, 1)`).

## Component States

### Indicator Box States

Each of the 5 indicator boxes (`PanelContainer` + `Label`) has two defined states:

| State | StyleBoxFlat bg_color | Label text color | Label outline | Trigger |
|-------|-----------------------|------------------|---------------|---------|
| Idle | `Color(0.10, 0.10, 0.10, 1)` | `Color(0.35, 0.35, 0.35, 1)` | None | Default; restored after fade-out |
| Active | Indicator-specific lit color (see table above) | `Color(1.0, 1.0, 1.0, 1)` | `outline_size = 1` | `activate()` called by `_on_hud_event` |

**Fade sequence (per indicator, independent):**
1. `activate()` called — set StyleBoxFlat to lit color, label text to white, outline on
2. Hold for **2.0 seconds** (Tween `tween_interval(2.0)`)
3. Fade `modulate.a` from `1.0` to `0.0` over **0.5 seconds** (`tween_property(node, "modulate:a", 0.0, 0.5)`)
4. On tween complete: reset StyleBoxFlat to idle color, label text to dim, outline off, `modulate.a = 1.0`

If `activate()` is called again while a fade is already running, kill the existing tween and restart from step 1 (re-light the indicator).

### Loop Label States

|-------|------|-------|
| Game start | `"Loop: 1"` | `Color(0.85, 0.85, 0.85, 1)` |

## Interaction Flow

Phase 7 UI is read-only from the player's perspective — no keyboard input targets the CarHUD. The interaction flow is event-driven only.

```
Game event fires on host
  → GameEvents.emit_hud.rpc("event_key")          # RPC broadcasts to all peers
  → All peers: GameEvents.hud_event signal fires
  → CarHUD._on_hud_event("event_key") called
  → _indicators["event_key"].activate()
      → StyleBoxFlat bg → lit color
      → Label text color → white
      → Tween: hold 2.0s → fade modulate.a to 0 over 0.5s
      → Tween complete: restore idle state
```

```
GameState.loop_number changes (via start_next_loop() — Phase 8 triggers)
  → CarHUD._process() polls GameState.loop_number each frame
      OR GameState emits a signal on change (planner decides pattern)
  → LoopLabel.text = "Loop: %d" % GameState.loop_number
```

```
Elite enemy spawn (host timer fires every 45–90s)
  → Host: $EnemySpawner.spawn.call_deferred({"type": "elite", ...})
  → _do_spawn_enemy() instantiates EliteEnemy scene
  → GameEvents.emit_hud.rpc("lidar") fires → LIDAR indicator activates on all screens
```

**No keyboard input map needed for CarHUD.** All interaction is signal-driven RPC.

**StyleBoxFlat setup for each indicator PanelContainer:**
- Create a `StyleBoxFlat` resource, set `bg_color = Color(0.10, 0.10, 0.10, 1)`, `corner_radius_*` all = 0 (flat)
- Apply via `add_theme_stylebox_override("panel", style_box)` in `CarHUD.gd _ready()`
- Store each StyleBoxFlat in a Dictionary keyed by event name so `activate()` can change `bg_color` without node lookups

**Indicator dictionary (in CarHUD.gd):**

```gdscript
var _indicators: Dictionary = {
    "ac":           {"panel": $..AcIndicator,          "label": $..AcLabel,          "lit": Color(0.1, 0.35, 0.85, 1)},

```
