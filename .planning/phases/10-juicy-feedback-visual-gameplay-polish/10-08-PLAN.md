Fix the confirmed live burn/slow status-visibility bug (ABIL-01) FIRST, then build the element-specific hit VFX (DMG-07) on top of the corrected sync state, and add the cosmetic enemy spawn-in telegraph (ABIL-06). This is the roadmap's wave-4 ordering: "fix the discovered host-only burn/slow visibility bug first … then build element-specific hit VFX … on top of the corrected sync state."

The status bug is real and verified: `Enemy.tscn`'s `SceneReplicationConfig` replicates only `position`/`current_hp`/`state` (properties/0-2, confirmed) — `modulate` and the status flags are NOT replicated, and `apply_burn()`/`apply_slow()`/`_tick_status_effects()` run only inside the host-only `_physics_process`, so burn/slow tints appear only on the host today. The fix adds two replicated booleans and moves the tint WRITE into `_process` (runs on every peer), while keeping the DoT/speed math host-only.

DMG-07 element hit VFX are burst-only (~0.4s, D-05) and read the newly-replicated flags: a burning enemy shows fire embers + an orange number, a slowed enemy shows ice shards + a light-blue number (D-02). Earth is the team-heal/shockwave element and leaves no per-hit enemy status, so earth-dealt basic hits render the neutral (white) burst/number here — its green is carried by the shared `element_color` map at earth's own ability sites; this mapping is dictated by the locked "no new synced state beyond the ABIL-01 fix" constraint, not a scope reduction.

Purpose: Correct a pre-existing multiplayer bug and layer element-readable combat feedback on top of it.
Output: Replicated is_burning/is_slowed + client-visible tint; fire/ice element hit VFX + element damage-number color; enemy materialize telegraph.
</objective>
