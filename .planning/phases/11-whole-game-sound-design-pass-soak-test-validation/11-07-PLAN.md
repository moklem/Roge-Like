## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| spawner-replicated _ready → deploy cue | Deploy cue runs in each peer's local `_ready`; no cross-peer call. |
| host authority → _on_pulse | Pulse cue is host-local behind the existing authority guard. |
| local UI input → menu cues | UI cues run on the local client only; menus are pre-session and non-networked. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-14 | D (DoS: audio spam) | HealDrone pulse cue | mitigate | `_on_pulse` fires on a fixed 3s Timer behind the host-authority guard; a client cannot invoke it. Playing host-local (no RPC) also removes any cross-peer amplification vector. |
| T-11-15 | D | UI/menu cues | accept | Menu/lobby handlers are local, pre-session UI with no multiplayer surface; cue rate is bounded by human click speed. Cues placed after early-returns prevent spamming a disabled/locked control. |
| T-11-16 | S/T | drone/orb/UI cue paths | accept | All cue paths are hardcoded in Sfx.gd; no peer data flows into them. No path-traversal surface, and no RPC is added by this plan (verified: HealDrone gains no `@rpc`). |
