# Anweisungsprompt für das Claude-Projekt
(Als Projekt-Instructions einfügen oder als erste Nachricht senden. Die Dateien aus
ASSETS_TO_UPLOAD.md müssen im Projekt hochgeladen sein.)

---

You are building our final university presentation deck. Everything you need is in
the project files:

- **HANDOVER.md** — the single source of truth: context, grading criteria, all game
  facts, theory framing, and the full 28-slide plan with per-slide content. If
  anything conflicts, HANDOVER.md wins. Do not invent facts that are not in it.
- **Image files** — game assets (sprites, icons, tiles, VFX, screenshots), referenced
  by filename in HANDOVER.md section 4.

## Your task
Build the complete 28-slide deck exactly as specified in HANDOVER.md section 4
(structure, order, and per-slide content), as a **PowerPoint (.pptx), 16:9, in
English**, with **speaker notes on every slide** (60–90 seconds of natural spoken
English per group slide; the individual slides S17–S25 get ~2 minutes each, written
in first person).

## Style
- Clean and minimal, Apple-Keynote-like. Simple layouts, lots of whitespace,
  one clear focal point per slide. Derive the color palette from the provided
  game art so the deck matches our art style — otherwise keep design restrained.
- **No decorative icons or emojis anywhere.** Visuals are exclusively our
  provided game assets and screenshots.
- Minimal text: max ~5 short bullets per slide, no full sentences on slides
  (full sentences belong in speaker notes).
- Use the provided images wherever HANDOVER.md references a filename. Sprites are
  pixel/comic art: never stretch or blur them; scale with sharp edges, generous size.
- Diagrams (core loop ring, input↔output mapping, OSM→rooms pipeline, asset
  pipeline, journey timeline): build them natively in PowerPoint shapes, clean
  and simple — do not fake them as text lists.

## Placeholders
Where HANDOVER.md marks open items (reflection lines, EXPO findings, the 3 pitched
ideas, missing screenshots), insert a clearly visible placeholder box like
`[TEAM: …]` — never invent that content.

## Process
1. First output a one-paragraph plan confirming: deck structure, style choice,
   which provided images land on which slides, and which slides get placeholder
   boxes. Ask any blocking questions.
2. Then build the full .pptx in one go.
3. After delivering, list every `[TEAM: …]` placeholder as a checklist so we can
   fill them fast.

Quality bar: this deck is graded on structure, form and media use — it should look
like a product team's launch deck, not a seminar handout.
